# 設計

## サイトの構成

| ページ           | パス                       | 内容                                      | 優先度 |
| ---------------- | -------------------------- | ----------------------------------------- | ------ |
| トップページ     | `/`                        | 最新記事一覧（新着順）                    | 高     |
| 記事ページ       | `/posts/[slug]`            | 個別記事ページ                            | 高     |
| アーカイブ（年） | `/archives/[year]`         | 指定年のすべての記事                      | 低     |
| アーカイブ（月） | `/archives/[year]/[month]` | 指定年月のすべての記事                    | 低     |
| カテゴリーページ | `/categories/[category]`   | 指定カテゴリーのすべての記事              | 低     |
| About ページ     | `/about`                   | 概要                                      | 低     |
| 検索結果ページ   | `/search?q=...`            | キーワードによる記事検索結果              | 中     |
| 管理画面         | `/admin`                   | Markdown/画像の投稿・編集画面（認証付き） | 高     |

## プロジェクト構成

**ブログ投稿**

### 記事

#### データ構造

| フィールド名   | 型        | 説明                         |
| -------------- | --------- | ---------------------------- |
| `id`           | string    | UUID                         |
| `title`        | string    | 記事タイトル                 |
| `slug`         | string    | URLスラッグ                  |
| `created_at`   | timestamp | 作成日時                     |
| `published_at` | timestamp | 公開日時                     |
| `updated_at`   | timestamp | 最終更新日時                 |
| `category`     | string    | カテゴリー                   |
| `content`      | string    | Markdownテキスト             |
| `isPublic`     | boolean   | 公開（そうでなければ下書き） |

**インデックス**

- `slug` にユニークインデックスを設定（記事のURLに使用するため）
- `published_at` にインデックスを設定（公開日時順での表示に使用するため）
- `category` にインデックスを設定（カテゴリー別記事の表示に使用するため）
- `isPublic` にインデックスを設定（公開記事と下書きを分けて取得するため）
